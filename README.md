# GaussianSplatterEffectsStudio

本地完整 3D 高斯泼溅特效与电影运镜工作台。程序直接读取 MipMap 导出的二进制 PLY，用完整椭球投影、透明合成、相机深度排序和球谐颜色渲染模型，再通过 GPU 生成动画并逐帧导出视频。

## 直接使用

双击 `dist/GaussianSplatterEffectsStudio.exe`。程序会启动本地服务并打开 Edge 独立应用窗口；制作期间请保留小型启动器窗口。

1. 导入 `gs.ply` 或 `ue/gs_full.ply`。
2. 选择高斯特效和电影运镜，调整强度、时长等参数。
3. 点击“播放动画”预览；时间轴可以拖动检查任意阶段。
4. 选择分辨率、帧率、码率和导出模式，然后点击“渲染视频”。程序会打开独立预览窗口；完成后可直接播放并点击“导出视频”。

导入后默认显示清晰的完整结束帧。“高斯尺寸”保持 `1.00` 时最接近原始模型。

## 手动视角

- 左键拖动：球面自由环绕，可连续跨越顶部和底部，不再受俯仰角极限卡住。
- 右键或中键拖动：平移观察中心。
- 滚轮：靠近或远离。
- 双击画布或点击“复位手动视角”：恢复默认构图。
- 播放期间开始手动操作会停在当前时间和当前机位，不会与自动运镜互相抢夺控制。
- “显示三维轨迹”会绘制蓝色轨迹、绿色起点、红色终点和黄色当前机位；录制时自动隐藏。
- 鼠标控制的工作视角与拍摄轨迹完全分离。鼠标环绕、平移和缩放不会修改最终视频的机位。

## 高斯特效

- 爆散聚合：粒子从四周和上方聚合成模型。
- 扫描显形：径向扫描波逐层还原高斯。
- 粒子雨：粒子由上向下落位。
- 旋涡聚合：绕模型旋转并收束。
- 晶格拼装：由规则空间晶格拼接回原形。
- 尘埃升腾：尘埃从下方上升并聚合。
- 能量脉冲：模型保持可见，能量波从内部向外扫过。
- 故障切片：分层横向错位，类似 glitch/transmission 动效。
- 光环扫描：霓虹环形波扫过模型，强化轮廓高光。
- 传送门旋入：粒子从环形传送门旋入并聚合。
- 烟花聚合：粒子从外侧烟花状收束回模型。
- 分层剥离：模型按高度层片剥离/归位。
- 呼吸辉光：模型保持可见，做柔和脉冲呼吸。

所有生成类特效在动画结束时恢复原始位置、椭球尺度、透明度和颜色。

## 电影运镜

- 静止、360° 环绕、旋转靠近、平滑推进、平滑拉远
- 升降揭示、横移弧线、8 字环绕、低机位英雄镜头

这些预设对应常见的 locked、orbit/arc、dolly、crane/pedestal 和 truck 类镜头，并针对单个 3DGS 主体做了平滑缓入缓出。运镜幅度为 `1.00` 时使用标准幅度。

运镜可以进一步自定义：

- 旋转中心 X/Y/Z：以模型包围盒中心为原点，数值单位是模型半径。
- 轨道轴：水平 Y 轴、垂直 X/Z 轴或用方位角与仰角定义任意倾斜轴。
- 顺/逆时针、0.25–3 圈旋转、起始方位角和起始俯仰角。
- 起始距离、结束距离倍率、镜头 FOV 和整体运镜幅度。
- 轨迹形状包括完整圆圈、指定角度圆弧、螺旋、直线、升降和 8 字轨迹。
- 轨迹圆心和视角跟随点相互独立。例如将圆心 Y 提高、跟随点留在模型中心，即可得到在物体上方盘旋并持续向下看的镜头。
- 起始/结束滚转角（Roll）控制摄像机绕自身视线轴旋转。两个角度相同是固定倾斜，不同则在运镜过程中连续滚转，同时保持相机位置与跟随点不变。
- 调整起始滚转角会自动预览动画开头视野；调整结束滚转角会自动预览动画结尾视野。
- `360° 环绕`、`旋转靠近`和`横移弧线`使用所选轨道轴；其他镜头仍围绕指定中心构图。

大部分数值参数同时支持滑条拖动和数字框输入。数字框可以直接键入精确数值，失焦或回车后会自动限制在该参数允许范围内。

## `gs.ply` 与 `gs_full.ply`

- `gs.ply`：完整位置、旋转、尺度、透明度和基础颜色（SH0），文件较小。
- `ue/gs_full.ply`：额外包含三阶球谐系数（SH3），能随视角还原颜色和高光变化，最高画质建议使用。
- 当前模型约 266 万个高斯；约 660 MB 的 `gs_full.ply` 会占用较多内存、显存和 CPU 排序时间。

## 画质建议

渲染质量提供“高质量、平衡、流畅预览”三档。修改渲染或输出参数后，状态会显示“尚未应用”；点击“应用当前渲染设置”后才会显示绿色的实际生效配置。开始导出前也会自动应用当前选择。

- 最接近网页预览：所见即所得实时录制、当前预览尺寸、30 FPS、12–25 Mbps。
- 120 FPS 选项已提供，但实时录制能否真正达到 120 FPS 取决于屏幕刷新率、浏览器录制器和当前模型渲染速度；常规显示器上更建议用 30/60 FPS。
- 常规成片：1080p、30 FPS、25 Mbps。
- 高质量：1440p、30 FPS、50 Mbps。
- 4K：先做短片测试；266 万高斯的 4K/60 FPS 会生成较大的中间帧数据。
- 高质量 MP4 导出：逐帧渲染固定时间码，再调用用户设备上的外部 FFmpeg 编码为 H.264 MP4。它不依赖浏览器实时录制器，适合 1440p/4K、60 FPS、最终成片和可拖动进度条的输出；导出时间会比实时录制更长。
- 所见即所得实时录制在 4K 下会自动避开 Edge 容易失败的 MP4/H.264 录制器，优先使用 WebM/VP9 或 VP8。若浏览器仍然只生成极小/损坏文件，程序会直接提示录制失败，不再导出打不开的视频。
- “所见即所得实时录制”会直接录制网页上的 canvas，观感最接近实时预览，文件也更容易被播放器顺滑解码；但如果当前实时渲染只有 51 FPS，选择 60 FPS 时输出也可能达不到严格 60 FPS。实时录制会优先使用 H.264 或 VP8，避免 VP9 编码造成额外卡顿。
- 选择“所见即所得实时录制”时，程序固定请求 60 FPS，帧率下拉框会自动锁定；只有“精确帧率逐帧导出”允许选择 24/30/60/120 FPS。
- 分辨率选择“当前预览尺寸”时，实时录制不会为了导出而强行重设画布大小，通常比固定 1080p/4K 更顺滑。
- “精确帧率逐帧导出”采用固定时间码逐帧 WebM：即使单帧渲染速度慢于实时播放，生成文件仍保持选择的 30/60 FPS；但它是全关键帧封装，播放器解码压力更大，某些播放器里可能不如实时录制顺滑。
- “高质量 MP4 导出”会优先尝试显卡 H.264 编码器 `h264_nvenc`，不支持时自动退回 `libx264`。输出默认写入 `+faststart` 索引，通常可以直接拖动进度条。
- 逐帧导出的 WebM 会写入 Cues 索引，通常可以拖动进度条。浏览器实时录制的 WebM 由 `MediaRecorder` 生成，某些播放器可能因为缺少完整索引而不能拖动；预览窗口会自动尝试修复时长元数据。
- 导出完成后，如果本机能找到 FFmpeg，程序会自动执行 `-c copy` 无损重封装，默认提供带 `_seekable` 的可拖动版本。该步骤只重写容器索引，不重新编码，因此不会改变原视频画质。
- 预览窗口显示的实时 FPS 只代表当前编辑/播放速度；最终视频应以导出模式为准。
- 逐帧导出时，码率选项会映射为 WebP/VP8 帧质量；更高码率会提高画质，也会增加 4K 导出时间和内存占用。

## 导出模式原理

软件目前提供三种导出模式。它们的目标不同，不是简单的“谁一定更好”，而是分别适合预览、精确帧率和最终成片。

### 所见即所得实时录制

原理：直接录制当前网页画布 `canvas.captureStream()`，再交给浏览器的 `MediaRecorder` 实时编码。

特点：

- 画面最接近你在软件里实时看到的预览，因为它录的就是当前画布。
- 导出速度最快，基本接近实时播放。
- 文件通常更容易被播放器顺滑解码。
- 缺点是受浏览器录制器、显卡驱动、分辨率和当前实时 FPS 限制。如果实时预览本身只有 51 FPS，那么导出请求 60 FPS 也可能出现轻微不均匀。
- 1440p/4K 时浏览器的 MP4/H.264 实时编码器可能不支持当前配置，所以程序会优先避开容易失败的 MP4/H.264，改用 WebM/VP8 或 VP9。

适合：

- 快速预览；
- 当前画面已经很流畅，并且只需要接近网页预览观感；
- 1080p 或当前预览尺寸导出。

不太适合：

- 严格固定 60/120 FPS；
- 4K 最终成片；
- 对进度条可拖动、封装兼容性要求很高的输出。

### 精确帧率逐帧导出

原理：按照所选帧率计算固定时间点，例如 8 秒 60 FPS 就渲染 480 帧；每一帧都由 GPU 渲染当前时间点，再封装成固定时间码 WebM。

特点：

- 帧数最准。即使单帧渲染慢于实时播放，最终文件仍然按所选 24/30/60/120 FPS 播放。
- 不依赖浏览器实时录制速度，不会因为预览时掉帧就少导出帧。
- WebM 会写入 Cues 索引，通常可以拖动进度条。
- 缺点是逐帧 WebM 的播放器解码压力可能更大；某些播放器看起来可能不如实时录制或 MP4 顺滑。
- 高分辨率时导出时间、内存占用都会增加。

适合：

- 检查动画时间点是否精确；
- 需要固定帧数，但不一定要求最终 MP4；
- 作为不依赖 FFmpeg 的备用导出方式。

不太适合：

- 最终公开视频；
- 4K/60 这种对播放器兼容性要求较高的成片。

### 高质量 MP4 导出

原理：先按固定时间码逐帧渲染，再把每帧发送给本地服务，由用户设备上安装的外部 FFmpeg 编码为 H.264 MP4。程序优先尝试显卡编码 `h264_nvenc`，如果不可用则自动退回 CPU 编码 `libx264`。输出会写入 `+faststart` 索引，方便播放器拖动进度条。

特点：

- 最适合最终成片，尤其是 1440p/4K、60 FPS。
- 不依赖浏览器实时录制器，因此不会遇到“浏览器编码器配置不支持”导致的坏 MP4。
- 固定帧率，画面节奏更稳定。
- MP4/H.264 兼容性更好，更适合上传、剪辑、分享和长期保存。
- 缺点是需要用户自行安装 FFmpeg；导出时间比实时录制长；高分辨率会产生较大的临时帧数据。

适合：

- 正式导出；
- 1440p/4K；
- 60 FPS；
- 希望视频可拖动进度条；
- 希望在常见播放器和剪辑软件里更稳定打开。

不太适合：

- 没有安装 FFmpeg 的环境；
- 只想快速看一眼效果的预览。

## 参数说明

### 渲染质量

渲染质量影响的是实时预览和导出时的高斯渲染方式，不只是视频压缩。

- 高质量：完整 SH、最高像素比、画面最清晰，适合最终检查和正式导出。缺点是 4K/大模型时压力最大。
- 平衡：保留较高画质，同时降低部分 GPU 压力，适合大多数编辑场景。
- 流畅预览：更重视交互帧率，适合调整运镜和特效。最终导出前建议切回“高质量”或“平衡”。

修改渲染质量后，界面会提示“尚未应用”。点击“应用当前渲染设置”后才会立刻更新；开始导出前也会自动应用当前设置。

### 分辨率

分辨率决定最终视频像素尺寸，也直接决定 GPU 渲染压力和编码压力。

- 当前预览尺寸：最接近当前窗口，通常最顺滑，适合实时录制。
- 1080p 横屏：兼容性最好，推荐日常测试和普通成片。
- 1440p 横屏：清晰度明显高于 1080p，适合高质量输出。
- 4K 横屏：细节最高，但渲染、读回、编码压力都很大，建议先短片测试。
- 竖屏分辨率：适合手机短视频平台，但同样会增加高分辨率导出的压力。

注意：4K 是否流畅，不只看电脑性能，还取决于导出路径。实时录制 4K 容易受浏览器编码器限制；高质量 MP4 模式更适合 4K。

### 帧率

帧率决定每秒包含多少帧。

- 24 FPS：电影感更强，文件更小，运动会更有跳帧感。
- 30 FPS：通用选择，导出速度和流畅度平衡。
- 60 FPS：更顺滑，适合旋转、飞入、粒子聚合等运动较多的镜头。
- 120 FPS：适合高刷展示或后期慢动作，但文件更大，导出时间更长，播放器和设备压力也更高。

在“所见即所得实时录制”中，软件固定请求 60 FPS，因此帧率下拉框会锁定。原因是实时录制的真实帧率由浏览器和实际渲染速度决定，开放多个帧率反而容易造成误解。  
在“精确帧率逐帧导出”和“高质量 MP4 导出”中，帧率会直接决定总帧数，例如：

- 8 秒 × 30 FPS = 240 帧；
- 8 秒 × 60 FPS = 480 帧；
- 8 秒 × 120 FPS = 960 帧。

### 码率

码率决定视频压缩时每秒可使用的数据量，通常越高画质越好、文件越大。

- 12 Mbps：适合快速预览或 1080p 轻量输出。
- 25 Mbps：推荐 1080p 成片。
- 50 Mbps：推荐 1440p 或高质量 1080p。
- 80 Mbps：推荐 4K 或复杂粒子动画。

不同导出模式里码率含义略有不同：

- 实时录制：传给浏览器 `MediaRecorder`，浏览器可能会根据自身编码器能力调整实际结果。
- 精确帧率逐帧导出：映射为 WebP/VP8 帧质量，更高码率会提高单帧质量。
- 高质量 MP4 导出：传给 FFmpeg 的 H.264 编码器，直接影响最终 MP4 的压缩质量和文件大小。

### 动画时长

时长决定整段动画播放多久，也会影响逐帧导出的总帧数。

更长的时长会让粒子聚合、扫描、运镜移动更慢更柔和；更短的时长会更有冲击力，但运动可能显得急促。

导出成本可以这样估算：

```text
总帧数 = 动画时长 × 帧率
```

例如 12 秒 4K/60 FPS 需要渲染 720 帧，明显比 8 秒 1080p/30 FPS 更慢。

### 高斯尺寸

高斯尺寸控制每个高斯椭球在屏幕上的整体缩放。

- `1.00`：最接近原始模型。
- 更小：画面更锐、更薄，但可能出现稀疏、破碎、空洞。
- 更大：画面更饱满、更柔和，但可能变糊，细节会被椭球重叠覆盖。

如果你觉得 `gs_full.ply` 也不如原来清晰，优先检查高斯尺寸是否被调大、渲染质量是否处于“流畅预览”、分辨率是否是当前预览尺寸而不是固定高分辨率。

### 特效强度

特效强度控制粒子偏移、扫描、爆散、辉光、切片等效果的幅度。

- 低强度：更像轻微动态修饰，模型主体稳定。
- 中等强度：适合大多数展示视频。
- 高强度：更有冲击力，但可能让模型短时间过于分散或画面噪点变多。

如果模型在动画中“散得太开”或“看不清主体”，优先降低特效强度，而不是降低分辨率。

### 运镜幅度

运镜幅度是对当前运镜预设的整体倍率。

- 小于 `1.00`：镜头更克制，更适合产品展示。
- `1.00`：预设标准幅度。
- 大于 `1.00`：镜头运动更明显，更有短视频冲击感，但也更容易晕或穿过模型。

运镜幅度不会改变鼠标手动视角。最终视频机位由轨迹参数决定，鼠标只是编辑时观察模型。

### 轨迹形状

轨迹形状决定相机实际怎么移动。

- 圆圈：围绕中心完整环绕，适合展示主体整体。
- 圆弧：只走一段弧线，适合从一个角度切到另一个角度。
- 螺旋：一边环绕一边升降或改变距离，视觉更有层次。
- 直线：适合推进、拉远或平移展示。
- 升降：强调高度变化，适合从底部揭示到顶部。
- 8 字轨迹：运动更复杂，适合艺术化展示，但参数过大时容易显得晕。

### 旋转中心与视角跟随点

旋转中心决定相机轨迹围绕哪里走；视角跟随点决定相机始终看向哪里。两者可以不同。

例如你想做“在物体上方盘旋并向下看”：

- 把旋转中心 Y 调高；
- 把视角跟随点留在模型中心或略低处；
- 轨迹选择圆圈或螺旋；
- 起始俯仰角设置为偏上方视角。

这样相机的运动中心在物体上方，但镜头视线仍会看向下方主体。

### 轨道轴、方位角和俯仰角

轨道轴决定环绕平面：

- 水平 Y 轴：常见水平环绕。
- 垂直 X/Z 轴：适合竖向翻转、上下绕行。
- 自定义轴：用方位角和俯仰角定义倾斜轨道，适合更电影化的斜向盘旋。

起始方位角决定镜头从模型哪个方向开始；起始俯仰角决定镜头从高处、平视还是低处开始。

### 起始距离与结束距离倍率

起始距离决定相机离模型多远；结束距离倍率决定动画结束时距离如何变化。

- 结束距离倍率小于 `1.00`：镜头逐渐靠近。
- 等于 `1.00`：距离基本不变。
- 大于 `1.00`：镜头逐渐拉远。

“旋转靠近”这类镜头通常会使用小于 `1.00` 的结束距离倍率。

### FOV

FOV 是镜头视场角。

- 小 FOV：类似长焦，透视压缩，主体更稳，背景运动感较弱。
- 大 FOV：类似广角，透视夸张，运动更有冲击力，但边缘变形更明显。

如果镜头靠近模型时画面太夸张，可以降低 FOV；如果想要速度感和空间感，可以提高 FOV。

### 滚转角 Roll

滚转角控制摄像机绕自身视线轴旋转，也就是画面倾斜角。

- 起始滚转角：动画开头的画面倾斜。
- 结束滚转角：动画结尾的画面倾斜。
- 两者相同：整个镜头保持固定倾斜。
- 两者不同：镜头移动过程中逐渐旋转画面。

调整起始滚转角会预览开头视野；调整结束滚转角会预览结尾视野。  
滚转不会改变相机所在位置，也不会改变视角跟随点，只改变画面绕中心的旋转角度。

### 循环预览

循环预览只影响播放预览，不改变导出文件本身。导出仍然按照设置的动画时长渲染一遍。

### 显示三维轨迹

显示三维轨迹用于编辑机位：

- 蓝色：完整轨迹；
- 绿色：起点；
- 红色：终点；
- 黄色：当前时间点的相机位置；
- 紫色：轨迹圆心；
- 青色：视角跟随点。

导出时轨迹会自动隐藏，不会出现在最终视频里。

## 推荐组合

如果你只是想快速看效果：

- 导出模式：所见即所得实时录制；
- 分辨率：当前预览尺寸或 1080p；
- 帧率：固定 60 FPS；
- 码率：12–25 Mbps；
- 渲染质量：平衡或流畅预览。

如果你想检查动画帧数是否准确：

- 导出模式：精确帧率逐帧导出；
- 分辨率：1080p；
- 帧率：30 或 60 FPS；
- 码率：25 Mbps。

如果你要正式出片：

- 导出模式：高质量 MP4 导出；
- 分辨率：1440p 或 4K；
- 帧率：60 FPS；
- 码率：50 Mbps（1440p）或 80 Mbps（4K）；
- 渲染质量：高质量；
- 先导出 2–3 秒短片确认画面和流畅度，再导出完整视频。

## 运行与编译

- Windows 10/11、Microsoft Edge、支持 WebGL2 的显卡。
- 启动器请求 Edge 使用 D3D11 高性能 GPU；最终显卡选择仍受 Windows 和显卡控制面板影响。
- 模型只在本机读取和渲染，不会上传网络。
- FFmpeg 不是本项目内置组件。若用户希望导出的浏览器录制 WebM 默认可拖动，或使用“高质量 MP4 导出”，需要在系统 PATH 中安装 FFmpeg，或把 `ffmpeg.exe` 放到程序同目录。程序仅作为外部命令调用它做无损重封装或视频编码。

安装 FFmpeg 的推荐方法：

```powershell
winget install --id Gyan.FFmpeg -e --source winget
```

安装完成后关闭并重新打开 GaussianSplatterEffectsStudio。若 `winget` 不可用，也可以手动下载 FFmpeg，把 `bin/ffmpeg.exe` 复制到 `GaussianSplatterEffectsStudio.exe` 同一文件夹。
